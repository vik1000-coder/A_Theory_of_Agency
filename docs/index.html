Role prompts are policy controls.

This project tests whether the same civic AI tasks behave differently when the system is assigned a different role: assistant, advocate, mediator, government information service, campaign aide, researcher, or news provider.

The problem

A civic AI product does not answer as a role-less model. It answers as a particular civic actor. The same lawful prompt can be reasonable for an advocate, inappropriate for a government-information service, and incomplete for a mediator.

The evaluation asks a more specific question than “is the model biased?” It asks whether assigned role changes who receives useful civic help, whether mirrored lawful viewpoints are treated symmetrically, and whether the output fits the role it claims to perform.

What counts as a failure?

One-sided refusal: the system refuses one lawful viewpoint but answers the mirrored lawful viewpoint.

Over-refusal: the system refuses a lawful civic task when refusal was not warranted.

Role miss: the answer is fluent but does not match the assigned role’s civic duty.

What we tested

The main evaluation holds the model, source packet, prompt, role-presentation mode, and judge fixed, then varies assigned role. That makes role a counterfactual variable rather than background wording.

Baseline

Five local models, seven roles, thirty civic prompts, two role-presentation modes.

Role policy

Same grid, plus explicit allowed help, refusal criteria, source rules, uncertainty language, escalation triggers, and viewpoint parity.

Targeted checks

Failure-heavy sample, policy ablations, harder mirrored stress prompts, and a regression gate.

Calibration

Primary LLM judge, alternate-judge samples, and a prepared two-rater human packet.

What we learned

The strongest signal is asymmetric civic service. The role-policy intervention helps the clearest failures, especially on mirrored pairs and targeted bad cases, but it is not a universal quality improvement.

What improved

The role policy roughly halves one-sided refusals on the full mirrored-pair grid and substantially improves the targeted failure sample. This is the project’s clearest positive result.

What did not fully improve

The full-grid aggregate refusal change is small, non-refusal quality drops slightly, and some roles or models move in the wrong direction. The policy is a useful control, not a cure-all.

Which policy pieces mattered?

The ablations remove one component at a time on the same 300-key targeted sample. Negative refusal deltas are good; positive role-fit and quality deltas are good.

How to read the claims

This is an evaluation and remediation project, not a claim that one prompt policy solves civic AI fairness. The right takeaway is operational: assigned role should be versioned, tested, and release-gated because it changes civic behavior.

The current results use LLM judges. Refusal labels are more stable across judges than role-profile scores. Human review has been designed but not imported, so role-fit claims should remain cautious until two-rater labels are summarized.

Current limits

Small local models, U.S. civic topics, static source packets, a limited prompt set, and no imported human labels yet.

The exploratory frontier run is shown separately in the repository and paper artifacts, but it is not pooled with the main evidence.
